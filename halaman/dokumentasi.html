Dokumentasi

Struktur Folder

  • gaya/ — file CSS
  • halaman/ — konten setiap halaman
  • rancangan/ — komponen layout
  • media/ — file media seperti foto/video
  • konfigurasi/ — tempat menulis konfigurasi rute
  • sistem/ — engine routing

Semua yang perlu kamu tahu untuk mulai pakai ARKAJS, dari instalasi sampai deploy.

Instalasi

Buat project baru dengan satu command, tidak perlu install ARKAJS secara global:

npx create-arkajs nama-project

Kalau argumen nama project tidak diisi, file akan dibuat di folder kerja saat ini:

npx create-arkajs

Setelah selesai, masuk ke folder project dan jalankan dengan live server, misalnya:

cd nama-project
npx serve
ARKAJS memakai fetch untuk mengambil file rute.json dan halaman HTML, jadi project ini harus dijalankan lewat server (live server/localhost), bukan dibuka langsung sebagai file index.html dari sistem berkas.

Membuat rute

Setiap halaman baru didaftarkan sebagai rute lewat generator bawaan:

node arka.js rute /nama-rute

Contoh, membuat halaman kontak:

node arka.js rute /kontak

Command ini akan menanyakan beberapa hal secara interaktif:

  • Judul halaman (dipakai sebagai document.title)
  • File header yang dipakai (default rancangan/header.html, bisa pakai file lain)
  • File konten utama, otomatis dibuat di halaman/nama-rute.html
  • File CSS yang dipakai (default gaya/gaya.css)

Setelah selesai, rute baru otomatis terdaftar di sistem/rute.json. Tinggal tambahkan link navigasinya secara manual, misalnya di rancangan/navbar.html:

<a href="/kontak">Kontak</a>

Membuat script

Untuk logic JavaScript per halaman atau per komponen, gunakan generator yang sama:

node arka.js nama-file

Contoh:

node arka.js navbar-toggle

Ini akan membuat file logika/navbar-toggle.js dengan template siap pakai, dan otomatis menambahkan import-nya ke logika/import.js supaya langsung ikut dimuat.

Pakai runDOM

Karena ARKAJS me-render konten lewat fetch (bukan langsung ada di HTML saat load), elemen di halaman bisa muncul belakangan. runDOM adalah wrapper kecil yang menunggu elemen tersebut ada sebelum menjalankan logic-nya, jadi aman dari race condition.

import { runDOM } from "../sistem/util.js"

runDOM('.tombol-kirim', (el) => {
  el.addEventListener('click', () => {
    // logic kamu di sini
  })
})

runDOM juga mendukung beberapa selector sekaligus dalam satu pemanggilan:

runDOM(['.tombol-kirim', '#form-kontak'],
    (el, selector) => {
  // dipanggil untuk masing-masing
    elemen yang ditemukan
})

Parameter ketiga (opsional) adalah batas waktu pencarian elemen dalam milidetik, default 10000 (10 detik). Kalau elemen tidak pernah muncul sampai batas waktu ini, akan muncul pesan di console.

Develop

Struktur folder utama yang akan sering kamu sentuh saat develop:

  • halaman/ — konten utama tiap rute (HTML)
  • rancangan/ — header, navbar, sidebar, footer
  • logika/ — file JavaScript per fitur, di-import lewat logika/import.js
  • gaya/gaya.css — semua styling, sudah punya variabel warna lewat custom property CSS
  • sistem/rute.json — daftar rute dan file yang dipetakan ke masing-masing rute
  • sistem/ — bagian inti router, tidak perlu diedit kecuali memang mau mengubah cara kerja routing

Jalankan project dengan live server apa saja saat develop, misalnya:

npx serve
# atau
npx http-server

Deploy

ARKAJS adalah single-page application yang memakai pushState untuk navigasi. Artinya, saat seseorang membuka langsung URL seperti /dokumentasi (bukan dari klik link di dalam aplikasi), server tujuan harus tetap mengarahkan permintaan itu ke index.html, bukan mengembalikan 404. Hosting statis yang dipilih wajib mendukung pengaturan rewrite/redirect semacam ini.

Cloudflare Pages (disarankan)

Cloudflare Pages mendukung SPA secara native lewat file _redirects. Buat file ini di root project:

/*  /index.html  200

Lalu deploy dengan menghubungkan repo GitHub ke Cloudflare Pages, atau lewat CLI:

npx wrangler pages deploy .

Hosting statis lain yang juga mendukung

LayananKonfigurasi rewrite
NetlifyFile _redirects sama seperti Cloudflare Pages
VercelFile vercel.json dengan rule rewrites ke /index.html
GitHub PagesTidak mendukung rewrite server-side; perlu trik 404.html sebagai fallback
Nginx (VPS sendiri)try_files $uri /index.html; di konfigurasi server block
Kalau hosting yang dipilih tidak punya cara mengarahkan semua rute ke index.html, refresh di halaman selain beranda akan menghasilkan 404. Untuk project ARKAJS, Cloudflare Pages atau Netlify adalah pilihan paling sederhana karena tinggal menaruh satu file _redirects.
